# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| v2.6    | ✅ Yes             |
| v2.5    | ⚠️ Security fixes only |
| < v2.5  | ❌ No              |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly:

1. **Do NOT** open a public issue
2. **Email** the maintainer or use [GitHub Security Advisories](https://github.com/nilesh-gore/system-update-toolkit/security/advisories/new)
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial assessment**: Within 1 week
- **Fix release**: As soon as possible, depending on severity

## Scope

These scripts run with elevated privileges on some platforms (`sudo` on Linux). We take security seriously and review all contributions for:

- Command injection vulnerabilities
- Unsafe file operations
- Privilege escalation risks
- Unvalidated user input

Thank you for helping keep System Update Toolkit secure! 🛡️
